# 2 Варианты использования

<a name="sign_in_as_guest"/>

## 2.1 Войти как гость

**Описание.** Вариант использования "Войти как гость" позволяет пользователю войти как гость, для просмотра каталога.  
**Предусловия.** Пользователь выбрал способ "Sign in as guest".  
**Основной поток.**
1. Приложение отображает окно для входа;
2. Пользователь выбирает способ входа "Sign in as guest";
3. Приложение присваивает пользователю статус "гость";
4. Приложение отображает главное окно с каталогом;
5. Вариант использования завершается.

**Постусловия.** Пользователь, вошедший как гость, может только просматривать каталог с снаряжением и не имеет доступа к добавлению товара в корзину и к самой корзине.

<a name="sign_in_to_your_account"/>

## 2.2 Войти в свою учётную запись

**Описание.** Вариант использования "Войти в свою учётную запись" позволяет пользователю войти в учётную запись.  
**Предусловия.** Пользователь выбрал способ "Sign in".  
**Основной поток.**
1. Приложение отображает окно входа в учётную запись;
2. Пользователь вводит свои данные;
3. Пользователь подтверждает ввод;
4. Приложение проверяет данные пользователя;
5. Приложение присваивает пользователю статус "зарегистрирован";
6. Приложение отображает главное окно;
7. Вариант использования завершается.

**Альтернативный поток А1.**
1. Приложение отображает сообщение о некорректных данных;
2. Вариант использования завершается досрочно.

**Постусловия.** Пользователь может просматривать каталог и добавлять товары в корзину, для дальнейшего их заказа.

<a name="sign_up"/>

## 2.3 Зарегистрироваться

**Описание.** Вариант использования "Зарегистрироваться" позволяет пользователю создать учётную запись.  
**Предусловия.** Пользователь выбрал способ "Sign up".  
**Основной поток.**
1. Приложение отображает форму регистрации;
2. Пользователь вводит данные, включая имя и пароль;
3. Приложение проверяет введённые данные на уникальность;
4. Приложение создает учётную запись;
5. Приложение присваивает пользователю статус "зарегистрирован";
6. Вариант использования завершается.

**Альтернативный поток А2.**
1. Приложение уведомляет пользователя о существующем аккаунте;
2. Приложение запрашивает другой ввод;
3. Возврат к основному потоку.

**Альтернативный поток А3.**
1. Пользователь закрывает окно регистрации
2. Приложение скрывает окно регистрации;
3. Вариант использования завершается досрочно.

<a name="view_catalog"/>

## 2.4 Просмотреть транзакции

**Описание.** Загружает список специальной экипировки и снаряжения.  
**Предусловия.** Пользователь вошёл в приложение как зарегистрированный пользователь или гость.  
**Основной поток.**
1. Приложение получает список специальной экипировки и снаряжения;
2. Приложение отображает специальную экипировку и снаряжение в виде таблицы;
3. Вариант использования завершается.

**Альтернативный поток А4.**
1. Приложение уведомляет пользователя об отсутствии товаров;
2. Вариант использования завершается.

<a name="view_equipment_info"/>

## 2.5 Просмотреть информацию о снаряжении

**Описание.** Отображает подробную информацию о снаряжении с его описанием.  
**Предусловия.** Пользователь нажал на иконку/картину/название снаряжения.  
**Основной поток.**
1. Приложение отображает описание и подробную информацию о специальной экипировке или снаряжении, выбранной пользователем;
2. Вариант использования завершается.

<a name="add_equipment_to_cart"/>

## 2.6 Добавить снаряжение в коризну

**Описание.** Добавляет выбранное пользователем снаряжение в корзину.  
**Предусловия.** Пользователь нажал на кнопку добавления снаряжения в корзину.  
**Основной поток.**
1. Приложение добавляет в корзину выбранное пользователем снаряжение для заказа;
2. Вариант использования завершается.

<a name="remove_equipment_from_cart"/>

## 2.7 Удалить снаряжение из корзины

**Описание.** Удаляет выбранное пользователем снаряжение из корзины.  
**Предусловия.** Пользователь нажал на кнопку удаления снаряжения из корзины.  
**Основной поток.**
1. Приложение удаляет из корзины выбранное пользователем снаряжение;
2. Вариант использования завершается.

<a name="create_order"/>

## 2.8 Произвести заказ

**Описание.** Оформляет заказ пользователя и отправляет запрос о заказе продавцу.  
**Предусловия.** Пользователь нажал на кнопку оформления заказа во вкладке корзины.  
**Основной поток.**
1. Приложение уведомляет пользователя об отправке запроса продавцу на заказ снаряжения;
2. Вариант использования завершается.

<a name="view_order_history"/>

## 2.9 Просмотреть историю заказов

**Описание.** Отображает историю о предыдущих заказах.  
**Предусловия.** Пользователь нажал на кнопку истории заказов.  
**Основной поток.**
1. Приложение отображает пользователю информацию о предыдущих заказах снаряжения;
2. Вариант использования завершается.

<a name="sign_out_of_your_account"/>

## 2.10 Выйти из учетной записи

**Описание.** Позволяет зарегистрированному пользователю выйти из учётной записи.  
**Предусловия.** Пользователь выбрал "Exit".  
**Основной поток.**
1. Приложение выходит из учётной записи;
2. Вариант использования завершается.
